**Step 3 (GREEN): Create src/auth/errors.ts**

Import `errors` from `jose`. Import `AuthError` from `./types.js`.

Implement `mapJoseError(err: unknown): AuthError` following the exact mapping from RESEARCH.md Pattern 3. Key behaviors:
- `JWTExpired` -> `{ status: 401, error: 'invalid_token', description: 'token expired' }`
- `JWTClaimValidationFailed` with `claim === 'aud'` -> description: 'invalid audience'
- `JWTClaimValidationFailed` with `claim === 'iss'` -> description: 'invalid issuer'
- `JWTClaimValidationFailed` with `claim === 'nbf'` -> description: 'token not yet valid'
- `JWTClaimValidationFailed` with other claims -> description: `claim validation failed: ${claim}`
- `JWSSignatureVerificationFailed` -> description: 'invalid signature'
- `JWKSTimeout` -> `{ status: 503, error: 'service_unavailable', description: 'unable to validate token: key service unavailable' }`
- `JWKSNoMatchingKey` -> description: 'no matching key found'
- Generic fallback -> `{ status: 401, error: 'invalid_token', description: 'token validation failed' }`

NOTE: `JWTExpired` extends `JWTClaimValidationFailed` in jose, so the `instanceof JWTExpired` check MUST come BEFORE `instanceof JWTClaimValidationFailed` to avoid catching expired tokens as generic claim failures.

Implement the three header builders:

```typescript
export function buildWwwAuthenticateNoToken(resourceMetadataUrl: string): string {
  return `Bearer resource_metadata="${resourceMetadataUrl}"`;
}

export function buildWwwAuthenticate401(
  resourceMetadataUrl: string,
  error: string,
  errorDescription: string,
): string {
  return `Bearer resource_metadata="${resourceMetadataUrl}", error="${error}", error_description="${errorDescription}"`;
}

export function buildWwwAuthenticate403(
  resourceMetadataUrl: string,
  requiredScopes: string[],
): string {
  return `Bearer resource_metadata="${resourceMetadataUrl}", error="insufficient_scope", error_description="insufficient scope", scope="${requiredScopes.join(' ')}"`;
}
```
