1. Create `src/routes/` directory.

2. Create `src/routes/mcp-routes.ts` -- Protected MCP routes as Fastify encapsulated plugin:

   Export an async function `protectedRoutes(fastify: FastifyInstance)` that:

   a. Adds a scoped `preHandler` hook that runs ONLY for routes inside this plugin:
      - Extract Bearer token from `request.headers.authorization` (strip "Bearer " prefix)
      - If no token: call `mapMissingTokenError()` from auth-errors.ts, log at warn level, send error response with correlation_id field, `return reply`
      - If token present: call Phase 4's `validateToken(request)` to validate
      - On validation success: set `request.user`, create child logger with `{ userId: user.oid, username: user.preferred_username }` via `request.log = request.log.child({...})`
      - On validation failure: call `mapJoseErrorToRfc6750()`, log at warn level ("Auth rejected"), log raw error at debug level, send RFC 6750 error response with `{ error, error_description, correlation_id: request.id }` JSON body and WWW-Authenticate header, `return reply`

   b. Registers `POST /mcp` route:
      - Creates StreamableHTTPServerTransport and McpServer per existing Phase 1 pattern
      - Wraps `transport.handleRequest()` in `requestContext.run()` with context containing `{ correlationId: request.id, userId: request.user?.oid, username: request.user?.preferred_username, log: request.log }`
      - This ensures tool handlers can access correlation ID and user identity via `requestContext.getStore()`

   c. Registers `GET /mcp/events` route (SSE endpoint):
      - Same auth protection via the scoped preHandler
      - Include correlation ID in the SSE stream: send an SSE comment line `: correlationId={request.id}` at the start of the stream
      - Wrap transport handling in requestContext.run() same as POST /mcp

   **Critical: In async Fastify hooks, ALWAYS `return reply` after `reply.send()` to stop hook chain execution (Pitfall 3 from research).**

   **Critical: Use `preHandler` (not `onRequest`) for auth -- body parsing must complete first for POST /mcp (Pitfall 1).**

3. Create `src/routes/public-routes.ts` -- Unprotected routes as Fastify plugin:

   Export an async function `publicRoutes(fastify: FastifyInstance)` that registers:

   a. `GET /` -- Server info endpoint:
      ```json
      {
        "name": "wikijs-mcp",
        "version": "2.0.0",
        "auth_required": true,
        "protected_resource_metadata": "{MCP_RESOURCE_URL}/.well-known/oauth-protected-resource",
        "endpoints": {
          "GET /": "Server info",
          "GET /health": "Health check (unauthenticated)",
          "POST /mcp": "MCP JSON-RPC endpoint (requires Bearer token)",
          "GET /mcp/events": "MCP SSE events (requires Bearer token)",
          "GET /.well-known/oauth-protected-resource": "RFC 9728 discovery (unauthenticated)"
        }
      }
      ```
      Per user decision: include auth_required field and link to protected_resource_metadata URL.

   b. `GET /health` -- Health check (move from server.ts, same logic):
      Returns `{ status: "ok"|"error", message: "..." }` after checking WikiJS connection.

   Note: GET /.well-known/oauth-protected-resource is already registered by Phase 3. Do NOT re-register it here. Just ensure it's outside the protectedRoutes plugin scope.

4. Restructure `src/server.ts`:

   a. Import `buildLoggerConfig` from `./logging.js` and spread into Fastify constructor:
      ```typescript
      const server = fastify({
        ...buildLoggerConfig(),
      });
      ```

   b. Add global `onRequest` hook to set X-Request-ID response header on EVERY response:
      ```typescript
      server.addHook("onRequest", async (request, reply) => {
        reply.header("x-request-id", request.id);
      });
      ```
      This runs before any other hooks, guaranteeing the correlation ID appears on 401s, 404s, and all errors (Pitfall 4 from research).

   c. Register plugins:
      ```typescript
      server.register(publicRoutes);
      server.register(protectedRoutes);  // Encapsulated scope -- auth hook only applies here
      ```

   d. Remove all existing inline route definitions (they move to route modules).

   e. Keep: dotenv config loading, WikiJsApi instantiation, Fastify listen/start logic.

   f. Remove `console.log` statements -- use `server.log.info()` instead (per decision to consolidate to pino).

   **Important:** Since Phases 1-4 have not been executed yet, the Phase 4 auth middleware (`validateToken`, `AuthenticatedUser`, `decorateRequest`) may not exist. If the import from `./auth-middleware.js` is not available at implementation time, create a TODO placeholder:
