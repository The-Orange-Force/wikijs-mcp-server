<objective>
Add a Zod default of `/app/instructions.txt` for `MCP_INSTRUCTIONS_PATH` in `src/config.ts`, so deployers running `docker-compose up` with a customized `instructions.txt` get their content in the MCP initialize response without setting any extra env var.

Purpose: Close the Docker flow gap identified in the v2.4 audit — the `docker-compose.yml` already mounts `./instructions.txt:/app/instructions.txt:ro` but the config had no default, silently ignoring the mount unless the deployer also set `MCP_INSTRUCTIONS_PATH=/app/instructions.txt` in `.env`.

Output: One changed line in `src/config.ts`, two updated test assertions in `tests/config.test.ts`, one added field in `makeTestConfig()`, one additional comment line in `.env.example`. Full suite green.
</objective>
