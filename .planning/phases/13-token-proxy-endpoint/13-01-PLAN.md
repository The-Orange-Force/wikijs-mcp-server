<objective>
Implement POST /token endpoint that proxies authorization_code and refresh_token grants to Azure AD, with scope mapping, AADSTS error normalization, and response passthrough.

Purpose: MCP clients need to exchange authorization codes for tokens and refresh expired tokens. The proxy handles Azure AD format differences transparently -- clients send bare MCP scopes and receive bare MCP scopes back, while Azure AD sees its required api:// format.

Output: `unmapScopes()` in scope-mapper.ts, `handleTokenRequest()` in token-proxy.ts, POST /token route in oauth-proxy.ts, plus unit and integration tests.
</objective>

    **File 1: Add unmapScopes() to src/oauth-proxy/scope-mapper.ts**

    Add below existing exports:
    ```typescript
    /**
     * Reverse-map Azure AD scopes back to bare MCP format.
     *
     * Strips the `api://{clientId}/` prefix from scopes in Azure AD's token
     * response so MCP clients see bare scope names (wikijs:read, not
     * api://clientId/wikijs:read). OIDC scopes (openid, offline_access) and
     * scopes with a different prefix pass through unchanged.
     */
    export function unmapScopes(scopeString: string, clientId: string): string {
      if (!scopeString) return scopeString;
      const prefix = `api://${clientId}/`;
      return scopeString
        .split(" ")
        .map((scope) => scope.startsWith(prefix) ? scope.slice(prefix.length) : scope)
        .join(" ");
    }
    ```

    **File 3: Create src/oauth-proxy/token-proxy.ts**

    This is the core business logic module. It receives parsed form body, config values, and an injected fetch function, and returns a result object with status and body.

    Structure:
    1. AADSTS_TO_OAUTH const lookup table (~20 entries from research AADSTS Mapping Table)
    2. GENERIC_DESCRIPTIONS: Record<string, string> mapping OAuth error codes to generic descriptions
    3. Zod schemas: `authCodeSchema` and `refreshTokenSchema` (from research Pattern 2)
    4. `normalizeAzureError()` function (from research Pattern 4)
    5. `buildAzureTokenBody()` function using URLSearchParams (from research Pattern 3) -- maps scopes via mapScopes, strips resource param, removes client_secret
    6. Main `handleTokenRequest()` function:

    ```typescript
    export interface TokenProxyContext {
      clientId: string;
      tokenEndpoint: string;
      fetch: typeof globalThis.fetch;
      log: { info: Function; warn: Function; error: Function };
    }

    export interface TokenProxyResult {
      status: number;
      body: Record<string, unknown>;
    }

    export async function handleTokenRequest(
      body: Record<string, string>,
      ctx: TokenProxyContext,
    ): Promise<TokenProxyResult>;
    ```

    handleTokenRequest flow:
    1. Extract grant_type from body. If missing or unsupported, return { status: 400, body: { error: "unsupported_grant_type", error_description: "..." } } immediately. Do NOT call fetch.
    2. Select Zod schema by grant_type. Parse body. On validation failure, return { status: 400, body: { error: "invalid_request", error_description: first Zod error message } }.
    3. Compare client_id to ctx.clientId. On mismatch, ctx.log.warn with the received client_id (not the full body). Return { status: 400, body: { error: "invalid_client", error_description: "unknown client_id" } }.
    4. Build outbound body via buildAzureTokenBody() -- strips resource, maps scopes, removes client_secret.
    5. Call ctx.fetch() with POST to ctx.tokenEndpoint, Content-Type: application/x-www-form-urlencoded, body as URLSearchParams.toString(), signal: AbortSignal.timeout(30000).
    6. On fetch error (network, timeout): ctx.log.error with error message (not full body). Return { status: 502, body: { error: "server_error", error_description: "Authorization server unavailable" } }.
    7. Check response Content-Type. If not application/json, return { status: 502, body: { error: "server_error", error_description: "Authorization server unavailable" } }.
    8. Parse response JSON (wrap in try-catch for malformed JSON). On parse error, same server_error.
    9. If response status >= 400: normalize via normalizeAzureError(). Return { status: response.status, body: normalized }.
    10. On success: if body has `scope` field, reverse-map via unmapScopes(). Return { status: response.status, body: azureBody }.

    Log on every request: grant_type and Azure AD response status (or "error" on failure). Log duration separately in route handler. NEVER log request body, tokens, or authorization codes.

    3. Add POST /token route inside the plugin:

    ```typescript
    fastify.post("/token", async (request, reply) => {
      const body = request.body as Record<string, string>;
      const startTime = Date.now();

      const result = await handleTokenRequest(body, {
        clientId: opts.appConfig.azure.clientId,
        tokenEndpoint: buildAzureEndpoints(opts.appConfig.azure.tenantId).token,
        fetch: fetchFn,
        log: request.log,
      });

      const duration = Date.now() - startTime;
      reply.header("X-Upstream-Duration-Ms", String(duration));
      reply.header("Cache-Control", "no-store");
      reply.header("Pragma", "no-cache");
      reply.header("Content-Type", "application/json");

      request.log.info({ grantType: body?.grant_type, upstreamStatus: result.status, durationMs: duration }, "token proxy request");

      return reply.status(result.status).send(result.body);
    });
    ```
