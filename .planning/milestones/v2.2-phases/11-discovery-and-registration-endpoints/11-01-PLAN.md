<objective>
Implement OAuth authorization server metadata (RFC 8414) and Dynamic Client Registration (RFC 7591) endpoints in a new Fastify plugin, register it in both production and test servers.

Purpose: MCP clients (Claude Desktop, Claude Code) need to discover the server's OAuth capabilities and register as clients before initiating authorization flows.
Output: Three public endpoints (`/.well-known/oauth-authorization-server`, `/.well-known/openid-configuration`, `POST /register`) returning static JSON, plus integration tests.

   ```
   issuer: appConfig.azure.resourceUrl

   registration_endpoint: `${appConfig.azure.resourceUrl}/register`

   grant_types_supported: ["authorization_code", "refresh_token"]
   code_challenge_methods_supported: ["S256"]
   ```

6. Register `POST /register`:

   - Return 201 status with: `{ client_id: appConfig.azure.clientId, token_endpoint_auth_method: "none", grant_types: ["authorization_code", "refresh_token"], response_types: ["code"] }`.

Import only: `FastifyInstance` type from "fastify", `AppConfig` type from "../config.js", `SUPPORTED_SCOPES` from "../scopes.js". All relative imports use `.js` extensions.
