6. **POST /mcp route** (TRNS-01):
   ```typescript
   server.post("/mcp", async (request, reply) => {
     const transport = new StreamableHTTPServerTransport({
       sessionIdGenerator: undefined, // stateless mode
     });

     reply.raw.on("close", async () => {
       await transport.close();
     });

     await mcpServer.connect(transport);
     await transport.handleRequest(request.raw, reply.raw, request.body);
   });
   ```
   CRITICAL: Do NOT call reply.send() or return a value -- the transport writes directly to reply.raw. If Fastify throws "Reply was already sent", add `reply.hijack()` before `handleRequest()`. Test without hijack first.

7. **GET /mcp route** (TRNS-02):
   Per MCP 2025-03-26 spec and research: in stateless mode, GET /mcp returns 405 Method Not Allowed.
   ```typescript
   server.get("/mcp", async (request, reply) => {
     reply.status(405).send({
       jsonrpc: "2.0",
       error: { code: -32000, message: "Method not allowed." },
       id: null,
     });
   });
   ```
   Note: The user's CONTEXT.md mentions "GET /mcp/events" but the research recommends using "GET /mcp" per the MCP 2025-03-26 spec. The route surface in CONTEXT.md also says "POST /mcp, GET /mcp/events" -- use GET /mcp per spec compliance. In stateless mode this returns 405 anyway, so no SSE handling is needed.

8. **GET /health route** (keep, unauthenticated):
   Keep the existing health check logic that calls `wikiJsApi.checkConnection()`.

9. **GET / route** (server info, updated):
   ```typescript
   server.get("/", async () => ({
     name: "wikijs-mcp",
     version: "1.3.0",
     endpoints: {
       "GET /": "Server info",
       "GET /health": "Health check",
       "POST /mcp": "MCP JSON-RPC endpoint",
       "GET /mcp": "MCP SSE endpoint (405 in stateless mode)",
     },
   }));
   ```

10. **Remove ALL individual REST tool routes**: Delete every `server.get("/get_page", ...)`, `server.get("/list_pages", ...)`, `server.post("/create_page", ...)` etc. There should be ZERO /get_page, /list_pages, /search_pages, /create_page, /update_page, /delete_page, /list_users, /search_users, /list_groups, /create_user, /update_user, /list_all_pages, /search_unpublished_pages, /force_delete_page, /get_page_status, /publish_page routes.
