  <behavior>
    FILT-03 / get_page blocked page:
    - Test: get_page with a page whose path is "Clients/AcmeCorp" returns `{ isError: true, content: [{ type: "text", text: "Error in get_page: Page not found. Verify the page ID using search_pages or list_pages." }] }`
    - Test: The error text is byte-identical to what the existing catch block produces for a genuinely absent page (exact string match)
    - Test: get_page with a non-blocked path returns normal page data (no filtering)
    - Test: get_page where getPageById returns null/undefined skips isBlocked check and returns data as-is

    SEC-01 / timing safety:
    - Test: getPageById is always called (and awaited) before isBlocked is invoked -- verify via mock call order (getPageById mock records call timestamp/order, isBlocked mock verifies getPageById was called first)
    - Test: For a blocked page, getPageById IS still called (not short-circuited)

    FILT-04 / search_pages filtering:
    - Test: search_pages with results containing a blocked page ("Clients/AcmeCorp") excludes it from the returned array
    - Test: search_pages with no blocked pages returns all results unchanged
    - Test: search_pages with ALL results blocked returns empty array
    - Test: totalHits is adjusted downward (e.g., raw totalHits=5, 2 pages filtered -> adjusted to 3). Note: totalHits is not currently serialized to client, but adjust for defense-in-depth per locked decision.

    FILT-05 / list_pages filtering:
    - Test: list_pages with results containing blocked pages excludes them from the returned array
    - Test: list_pages with no blocked pages returns all results unchanged

    SEC-02 / audit logging:
    - Test: When get_page encounters a blocked page, a warn-level log entry is emitted with `{ toolName: "get_page", userId, username, gdprBlocked: true }` and message "GDPR path blocked"
    - Test: When search_pages filters N blocked pages, N individual warn log entries are emitted (one per blocked page)
    - Test: When list_pages filters blocked pages, warn log entries are emitted per blocked page
    - Test: Log entries do NOT contain the page path or any path-derived content
  </behavior>
  <implementation>
    Add to src/mcp-tools.ts:

    1. Import `isBlocked` from `./gdpr.js` and `requestContext` from `./request-context.js` (requestContext may already be indirectly available but import explicitly for the helper).

    2. Add `logBlockedAccess(toolName: string): void` helper function at the top of the file (inside or outside `createMcpServer` -- outside is cleaner since it doesn't need `wikiJsApi`):
       ```typescript
       function logBlockedAccess(toolName: string): void {
         const ctx = requestContext.getStore();
         ctx?.log.warn(
           { toolName, userId: ctx.userId, username: ctx.username, gdprBlocked: true },
           "GDPR path blocked",
         );
       }
       ```
       IMPORTANT: Do NOT add `correlationId` explicitly -- it is already bound to the pino child logger automatically (Pitfall 6 from RESEARCH.md).

    3. In `get_page` handler (after `const page = await wikiJsApi.getPageById(id);` on current line 65), add:
       ```typescript
       if (page?.path && isBlocked(page.path)) {
         logBlockedAccess(TOOL_GET_PAGE);
         return {
           isError: true,
           content: [{
             type: "text" as const,
             text: "Error in get_page: Page not found. Verify the page ID using search_pages or list_pages.",
           }],
         };
       }
       ```
       This MUST be after the `getPageById` call completes (SEC-01). The guard `page?.path` handles null/undefined page objects safely.

    4. In `list_pages` handler (after `const pages = await wikiJsApi.listPages(...)` on current line 123-127), replace direct serialization with:
       ```typescript
       const filtered = pages.filter(p => {
         if (isBlocked(p.path)) {
           logBlockedAccess(TOOL_LIST_PAGES);
           return false;
         }
         return true;
       });
       ```
       Then serialize `filtered` instead of `pages`.

    5. In `search_pages` handler (after `const result = await wikiJsApi.searchPages(query, limit)` on current line 178), add:
       ```typescript
       const originalLength = result.results.length;
       const filtered = result.results.filter(p => {
         if (isBlocked(p.path)) {
           logBlockedAccess(TOOL_SEARCH_PAGES);
           return false;
         }
         return true;
       });
       result.totalHits -= (originalLength - filtered.length);
       ```
       Then serialize `filtered` instead of `result.results`.
