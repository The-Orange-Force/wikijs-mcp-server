<objective>
Align scope notation between the discovery endpoint and auth middleware so that scopes are consistent across the entire server.

Purpose: The discovery endpoint (via src/scopes.ts) currently serves dot-notation scopes ("wikijs.read") while the auth middleware independently defines colon-notation scopes ("wikijs:read"). A client following the RFC 9728 discovery document would request the wrong scopes from Azure AD and be rejected with 403. This plan fixes the mismatch by standardizing on colon notation (the OAuth 2.0 / Azure AD convention) with a single source of truth.

Output: All scope strings use colon notation from src/scopes.ts; middleware imports from scopes.ts; all tests updated and passing.
</objective>
