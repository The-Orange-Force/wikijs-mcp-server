<feature>
  <name>redactContent -- GDPR marker-based content redaction</name>
  <files>src/gdpr.ts, src/__tests__/gdpr.test.ts</files>
  <behavior>
    REDACT-01 -- Single marker pair:
    - Input: "Hello <!-- gdpr-start -->SECRET<!-- gdpr-end --> World"
    - Output: content = "Hello [placeholder] World", redactionCount = 1, warnings = []

    REDACT-02 -- Multiple marker pairs:
    - Input: "A <!-- gdpr-start -->X<!-- gdpr-end --> B <!-- gdpr-start -->Y<!-- gdpr-end --> C"
    - Output: content = "A [placeholder] B [placeholder] C", redactionCount = 2, warnings = []

    REDACT-03 -- Exact placeholder text:
    - Each redacted block replaced with: "[lock-emoji PII redacted -- consult the wiki directly for contact details]"
    - Use exported REDACTION_PLACEHOLDER constant in both implementation and tests

    REDACT-04 -- Unclosed start marker (fail-closed):
    - Input: "Public <!-- gdpr-start -->Secret to end"
    - Output: content = "Public [placeholder]", redactionCount = 1, warnings = [unclosed warning]

    REDACT-05 -- Malformed marker warnings:
    - Unclosed gdpr-start: warning with message, pageId, path
    - Orphaned gdpr-end (no preceding start): warning with message, pageId, path
    - Input: "<!-- gdpr-end --> text": content unchanged (end marker left in place), warnings = [orphaned warning]

    REDACT-06 -- Case and whitespace tolerance:
    - "<!-- GDPR-START -->X<!-- GDPR-END -->" matches (uppercase)
    - "<!--gdpr-start-->X<!--gdpr-end-->" matches (no spaces)
    - "<!--  gdpr-start  -->X<!--  gdpr-end  -->" matches (extra spaces)
    - "<!-- Gdpr-Start -->X<!-- Gdpr-End -->" matches (mixed case)

    Edge cases:
    - Null/undefined/empty content: returns { content: "", redactionCount: 0, warnings: [] }
    - Content with no markers: returns content unchanged, count 0, no warnings
    - Multiline content between markers: correctly redacted (markers span lines)
    - Nested start markers: inner start is just redacted content (first-start to first-end pairing)
    - Content that is ONLY markers: replaced entirely with placeholder
  </behavior>
  <implementation>
    Add to existing src/gdpr.ts (preserving isBlocked):

    1. Export REDACTION_PLACEHOLDER constant with exact text from REDACT-03
    2. Export RedactionWarning interface: { message: string; pageId: number; path: string }
    3. Export RedactionResult interface: { content: string; redactionCount: number; warnings: RedactionWarning[] }
    4. Export redactContent(content: string, pageId: number, path: string): RedactionResult

    Function uses two-pass regex approach (per RESEARCH.md):
    - Pass 1: Replace matched pairs with non-greedy regex /<!--\s*gdpr-start\s*-->[\s\S]*?<!--\s*gdpr-end\s*-->/gi
    - Pass 2: Replace unclosed start markers with greedy regex /<!--\s*gdpr-start\s*-->[\s\S]*/gi (fail-closed)
    - Pass 3: Detect orphaned end markers /<!--\s*gdpr-end\s*-->/gi for warning generation only

    CRITICAL: Create regex objects INSIDE the function to avoid lastIndex state issues with the g flag.
    CRITICAL: All imports must use .js extensions (NodeNext module resolution).
    CRITICAL: Preserve existing isBlocked() function unchanged.
  </implementation>
</feature>
