# Roadmap Archive: WikiJS MCP Server — Docker Deployment (v2.1)

**Archived:** 2026-03-25
**Status:** Complete (1/1 phases, 1/1 plans)
**Core Value:** Only Azure AD-authenticated colleagues can invoke MCP tools against the company WikiJS instance

## Overview

This milestone packaged the MCP server as a Docker container for self-hosted deployment behind Caddy. A two-stage Dockerfile compiles TypeScript and produces a slim runtime image, docker-compose.yml joins the container to caddy_net with no host port exposure, and HEALTHCHECK validates the /health endpoint.

## Accomplishments

1. **Multi-stage Dockerfile** — node:20-slim builder compiles TypeScript, strips .map/.d.ts; runtime stage copies only dist/ and production deps
2. **docker-compose.yml** — Single-service deployment on caddy_net, .env file for config, restart: unless-stopped
3. **.dockerignore** — Excludes dev artifacts, .env, node_modules from build context
4. **Docker HEALTHCHECK** — Uses `node -e` against /health (no curl/wget in slim image)
