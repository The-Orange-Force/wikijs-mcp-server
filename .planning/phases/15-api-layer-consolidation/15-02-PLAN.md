  <files>src/api.ts, tests/api.test.ts</files>
  <behavior>
    - searchPages("test query") with all results resolvable via singleByPath returns { results: [pages with real DB IDs], totalHits: N }
    - searchPages with empty results returns { results: [], totalHits: 0 }
    - When singleByPath fails for some results, those results are resolved via pages.list cross-reference by path
    - When singleByPath fails for ALL results, a single pages.list batch call resolves them
    - Results that cannot be resolved by either method are dropped from results array
    - A warning is logged for each dropped result containing path and original search index ID
    - The totalHits value comes from the Wiki.js search response (not the resolved count)
    - singleByPath calls are made in parallel (Promise.allSettled), not sequentially
    - Only ONE pages.list fallback call is made regardless of how many results need fallback
    - The limit parameter caps the number of search results processed (sliced before resolution)
    - resolvePageByPath passes both path and locale to the singleByPath GraphQL query
  </behavior>
  <action>
**src/api.ts changes:**

1. Add import at top: `import { requestContext } from "./request-context.js";`
2. Add import: `import { ClientError } from "graphql-request";`
3. Import the PageSearchResult type: update the import from types.ts to include `PageSearchResult`.

4. Add private method `resolvePageByPath(path: string, locale: string): Promise<WikiJsPage>`:
   - GraphQL query: `pages { singleByPath(path: ..., locale: ...) { id path title description isPublished createdAt updatedAt } }`
   - No content field (search results are metadata-only, use getPageById for full content)
   - Returns `response.pages.singleByPath`

5. Add private method `resolveViaPagesList(unresolved: Array<{ path: string; locale: string; searchId: string }>): Promise<{ resolved: WikiJsPage[]; dropped: typeof unresolved }>`:
   - GraphQL query: `pages { list(limit: 500, orderBy: UPDATED) { id path title description isPublished createdAt updatedAt } }`
   - Build `Map<string, WikiJsPage>` keyed by path
   - For each unresolved item, look up by `item.path` in the map
   - Return resolved pages and still-dropped items separately
   - Limit of 500 per Claude's discretion (covers typical corporate wikis)

6. Replace `searchPages(query: string, limit: number = 10): Promise<PageSearchResult>`:
   - Step 1: Execute `pages.search` GraphQL query (same as existing, returns id/path/title/description/locale + totalHits)
   - Slice results to `limit` before processing
   - If no results, return `{ results: [], totalHits }`
   - Step 2: Call `resolvePageByPath` for each result in parallel via `Promise.allSettled`
   - Collect fulfilled results into `resolved` array, rejected into `unresolved` array (with path, locale, searchId)
   - Step 3: If `unresolved.length > 0`, call `resolveViaPagesList(unresolved)` once
   - Merge newly resolved into the resolved array
   - Step 4: Log a warning for each still-dropped result using `requestContext.getStore()?.log.warn()`
   - Step 5: Return `{ results: resolved, totalHits }`

**Key implementation details:**
- Use `JSON.stringify()` for embedding query and path values in GraphQL strings (existing pattern)
- For error detection in singleByPath: let it throw naturally (graphql-request throws on errors). Promise.allSettled catches it.
- Log a consolidated warning if ALL singleByPath calls fail with what looks like a permission error (check `ClientError` + message contains "Forbidden" or "ERR_FORBIDDEN"). This helps admins know to check token permissions.
