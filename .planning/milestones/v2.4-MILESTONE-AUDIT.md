### Phase 19: Instructions Loading (decisions + pre-existing)

5. **Pre-existing: WikiJsApi instantiated twice in server.ts** — one instance in `buildApp` (used for tools), one in `start()` (used for health check only). The second instance is discarded after the connection check. Not introduced by v2.4.

## E2E Flow Traces

**Flow 1: MCP initialize with default instructions (local dev)**
`MCP client POST /mcp` → `authPlugin validates Bearer token` → `protectedRoutes handler` → `createMcpServer(wikiJsApi, DEFAULT_INSTRUCTIONS)` → `McpServer({ instructions })` → initialize response includes 5-topic guidance ✓

**Flow 2: MCP initialize with file-loaded instructions**
`start()` → `loadInstructions(config.instructionsPath)` reads file → `buildApp(config, undefined, fileContent)` → instructions threaded through → initialize response includes file content ✓

**Flow 3: File missing / fallback**
`loadInstructions(path)` catches ENOENT/EACCES → `console.warn(...)` → returns `DEFAULT_INSTRUCTIONS` → server starts normally ✓

**Flow 4: Docker deploy with custom instructions**
`docker-compose up` → `instructions.txt` mounted at `/app/instructions.txt` → Zod default `'/app/instructions.txt'` activates without any extra env var → `loadInstructions('/app/instructions.txt')` reads file → initialize response with custom content ✓ *(Phase 21 gap closure)*
